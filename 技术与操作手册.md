# 浙江省·杭州市领导留言板实时监测大屏 — 技术与操作手册

> 版本：2026-06　|　适用：运维、二次开发、迁移交接

---

## 1. 系统概述

一块实时大屏，自动采集**人民网领导留言板**中杭州市及 8 个主城区的群众留言，做统计、地理落点、群众满意度、社会风险研判，并以"科技感大屏"呈现，每 10 分钟自动更新。

- **采集范围**：11 个版块 = 杭州市 + 市委书记 + 市长 + 上城/拱墅/西湖/滨江/萧山/余杭/临平/钱塘 8 区。
- **更新频率**：每 10 分钟（GitHub Actions 定时）。
- **成本**：全部免费档（GitHub Actions 公开仓库不限分钟、Vercel Hobby、高德/DeepSeek 免费额度）。
- **线上地址**：https://liuyanzhejiang.vercel.app/

## 2. 总体架构

```
                 ┌────────────────────── GitHub Actions（每10分钟）──────────────────────┐
人民网留言板 API ─▶ zj_live.py(抓取/聚合) ─▶ data.json                                      │
                 │   geocode.py(实体→高德地理编码) ─▶ geo.json   [密钥 AMAP_KEY]            │
                 │   risk.py(规则+DeepSeek研判) ─▶ risk.json      [密钥 DEEPSEEK_API_KEY]   │
                 └──────────────── 强制推送到 GitHub 的 data 分支 ◀───────────────────────┘
                                            │
              Vercel(部署 main 分支) ── /api/data /api/geo /api/risk 读取 data 分支 ──▶ index.html 大屏
```

- **data 分支 = 数据库**：每轮把 3 个 json + 2 个缓存压成单提交强推，文件内部累积、历史不膨胀。
- **main 分支**：代码 + 种子 data/geo/risk.json（首次启动与 Vercel 兜底用）。
- **关键**：仓库必须 **Public**（raw 读取 + Actions 免费）；`vercel.json` 关闭 data 分支自动部署。

## 4. 数据来源 API（人民网留言板）

**接口**：`POST https://liuyan.people.com.cn/pro-dfbbs-front/threads/queryThreadsList`

| 参数 | 说明 |
|---|---|
| `fid` | 版块 ID（见 §8 版块表） |
| `lastItem` | **游标**=上一批最后一条的 `tid`；首次传 0（不是页码！） |
| `pageSize` | 每页条数（实际约返回 10 条） |

**返回字段**（取用的）：`tid`(帖ID,随时间增大)、`dateline`(时间戳)、`subject`(标题)、`content`(正文)、
`domainName`(领域)、`typeName`(诉求性质)、`stateInfo`(状态)、`answerId`/`answerContent`/`answerOrganization`(回复)、
`gradeManner`/`gradeSpeed`(态度/速度评分 1–5)、`favNum`(点赞)、`ip`(打码)、`nickName`。

**反爬要点**：连续快速请求会 403/429。脚本用 `requests.Session` + 先 GET 版块页拿 Cookie(warmup)，每版块只翻 1 页，
请求间隔+抖动，403/429 时重新 warmup+长等待。迁移到境外服务器仍可用（首次跑日志确认连通）。

## 5. 数据管线

### 5.1 `zj_live.py`
- `--mode seed --csv <csv> --out data.json`：用历史 CSV 建底图，并设各版块 watermark(已见最大 tid)。
- `--mode update --data data.json --out data.json [--delay 1.5] [--pages 1]`：只抓 tid>watermark 的新留言，去重累积。
- 去重：每版块 watermark（批前快照 + 批内 id 集合）。状态为**入库快照**（不回刷）。

### 5.2 `geocode.py`（需 `AMAP_KEY` 环境变量）
- 实体提取：正则抽小区/路/公司/医院/学校等地点实体（`--dry-run` 实测覆盖率 72%）。
- 地理编码：`GET /v3/geocode/geo`(地址=区名+实体, city=杭州) → 经纬度+所属街道；无街道则 `/v3/geocode/regeo` 逆编码补。
- `--mode full`：历史全量（缓存命中后秒级）；`--mode live`：只编码 data.json recent 中 tid>水位 的新留言。
- 缓存 `.geocache.json`：`"区|实体"→[lng,lat,街道]`，不重复请求。

### 5.3 `risk.py`（需 `DEEPSEEK_API_KEY`）
- 读 data.json 的 `pending` 未办结池 → 规则打分(§7 风险) + 聚类(同小区/公司 或 同区+议题) → 风险点。
- 对 全局TOP14 + 每区最高点 调 `deepseek-v4-pro` 出 `{level, judge, advice}`（缓存 `.riskcache.json`）。
- 久远留言提示词标注"历史积压"，避免按当前突发措辞。

## 6. 数据文件 Schema

### data.json
```
updated, source, watermark{fid:tid}, totals{all,replied},
byMonth{YYYY-MM:n}, byDomain{领域:n}, byStatus{状态:n}, byType{性质:n},
byForum{版块:{count,fid,level}}, byDistrict{区:n},
sat{mSum,mCnt,sSum,sCnt,dist{1..5}},                 # 全市满意度
kw{词:次},                                            # 热词(TOP400)
districts{区:{count, byMonth, sat, byDomain, byStatus, byType, byStreet, kw, low[]}},
low[≤80]{tid,title,content,date,forum,district,gm,gs,status},   # 全局低分(≤2★)
pending[≤400]{tid,title,content,date,forum,district,domain,type,status},  # 未办结池
recent[≤600]{id,tid,date,fid,forum,level,district,title,content,domain,type,status,hasReply,replyOrg,likes,ip,nick,gManner,gSpeed}
```
### geo.json
```
updated, wm{fid:tid},
districts{区:{ geo:已编码数, byStreet{街道:量}, streetSat{街道:{s态度和,c参评数}},
               streetGeo{街道:[lng,lat]质心}, points[[lng,lat]热力点] }}
```
### risk.json
```
updated, counts{红,橙,黄}, pendingTotal,
points[]{ key, level(红/橙/黄), topic, area(区/版块), object, count, days(最久距今天数),
          fresh(近1年有新留言), score, signals[命中类别], judge(研判), advice(应对),
          samples[{title,date,forum,content}] }
```

## 7. 指标字典（每个指标的定义与算法）

| 指标 | 定义 / 算法 | 来源字段 |
|---|---|---|
| 累计留言 | 收录总条数 `totals.all` | 全部抓取 |
| 今日新增 | recent 中日期=今天的条数 | recent.date |
| 本月新增 | `byMonth[本月]` | dateline→月 |
| 办结率 | 已办结/总数；已办结=状态∈{已办理,已回复,已解决,办结,已办结} 或 hasReply | stateInfo/answerId |
| 群众满意率 | 态度评分 4–5★ 占比 = (dist4+dist5)/参评数 | gradeManner |
| 满意度态度/速度均分 | gradeManner/gradeSpeed 的均值(参评者) | gradeManner/Speed |
| 评分分布(1–5★) | 各星级计数 `sat.dist` | gradeManner |
| 各区满意率 | 同上，按区 `districts[区].sat` | — |
| 诉求领域分布 | 按 domainName 计数(TOP10) | domainName |
| 诉求性质 | 按 typeName 计数(投诉求助/咨询/建言…) | typeName |
| 办理状态 | 按 stateInfo 计数(待回复/处理中/已办理…) | stateInfo |
| 月度趋势 | 按 年-月 计数；环比=最近完整月/上月，同比=去年同月 | dateline |
| 诉求热词 | jieba 分词标题去停用词，累计词频(TOP400)；各区热词去本区名 | subject |
| 区内街道·留言量 | 地理编码归属街道计数 `byStreet` | geocode |
| 区内街道满意度 | 该街道留言态度均分(参评≥3) `streetSat` | geocode+gradeManner |
| 街道覆盖率 | 已地理编码数/该区留言数 ≈ 60–70% | geo/count |
| 社会风险点 | 见 §下「风险口径」 | pending |
| 低分督办 | 态度或速度被打 1–2★ 的留言 `low` | gradeManner/Speed |

**风险口径（信访研判维度，规则可调，在 `risk.py` 的 `RISK_CATS`）**：
群体性(集体/多人/业主/联名…5分)、欠薪讨薪(5)、房产烂尾(4)、征地拆迁(4)、极端情绪(轻生/走投无路…6)、
公共安全(危房/燃气/坍塌…4)、激化诉诸(上访/曝光/起诉/多次反映…3)、环境污染(2)；叠加久拖(>30/90天)。
分级红/橙/黄(DeepSeek 可覆盖)。`fresh`=近1年有新留言→现行，否则历史积压。

## 8. 数据范围与口径

- **版块(11)**：杭州主城区口径：146 杭州市 / 1007 市委书记 / 1008 市长 /
  4171上城 4170拱墅 4174西湖 4175滨江 4176萧山 4177余杭 5193临平 5194钱塘。
- **时间跨度**：月度趋势约 2007→今（市级版块历史长；区级版块多为 2021 年后开通）。
- **满意度**：约 49% 留言有评分（办结并经群众评价的才有）。
- **未办结池**：取最新 ~400 条待回复/处理中。**注意**：主城区近期诉求大多已快速办结，未办结主要是
  2018–2022 历史积压，故风险多标"历史积压"；真正"现行"风险随 live 新留言流入。
- **状态为入库快照**：老留言被收录时是什么状态就记什么，不回头刷新（"距今天数"是留言距今，不等于真实积压时长）。
- **街道**：数据源无街道字段、无街道边界，街道归属靠正文实体+高德地理编码推断，覆盖约 60–70%（抽样参考）。

## 9. 后端读取 API（Vercel 函数）

| 路由 | 返回 | 机制 |
|---|---|---|
| `/api/data` | data.json | 读 `raw.githubusercontent.com/<owner>/<repo>/data/data.json`，失败回落部署内 `./data.json` |
| `/api/geo` | geo.json | 同上 |
| `/api/risk` | risk.json | 同上 |

- owner/repo 由 Vercel 系统变量 `VERCEL_GIT_REPO_OWNER/SLUG` 自动推断；可用环境变量 `DATA_URL/GEO_URL/RISK_URL` 覆盖。
- 边缘缓存 `s-maxage=120`（约 2 分钟）。强推 data 分支后，线上约 1–3 分钟转新（raw CDN + 边缘缓存）。

## 10. 前端结构（index.html，1920×1080 自适应）

- 顶栏：标题/时钟/累计/LIVE。左列：KPI(今日/本月/办结率/满意率)、群众满意度仪表+评分分布、办理状态+诉求性质双环。
- 中列：杭州主城区热力地图(轮播聚焦时缩为左上角小图+红点)、各区满意率排行+区内街道满意度。
- 右列：区内街道·留言量、诉求热词云、诉求领域分布。底栏：最新诉求滚动。
- **轮播**(5.5s)：8 区逐个 + 全市收尾；联动满意度/领域/热词/状态/性质/排行/趋势随当前区切换；
  中间地图下钻为该区"发光多边形+街道气泡"，右上弹出该区详情框(危机变红，含研判+应对+相关留言)。
- 点击区县 → 完整对比弹窗。

## 11. 自动化（.github/workflows/crawl.yml）

每 10 分钟（`cron: */10`，亦可 Actions 页手动 Run）：检出代码 → 装依赖 → 从 data 分支拉 data/geo/risk/缓存
(无则用种子) → `zj_live update` → `geocode live`(有 AMAP_KEY) → `risk`(有 DEEPSEEK_API_KEY) → 强推 data 分支。
> GitHub 免费版定时高峰期可能延迟/偶尔跳过；仓库 60 天无活动会暂停定时（本场景够用）。

## 12. 部署步骤

1. 新建 **Public** GitHub 仓库，推送本目录。
2. Vercel → Add New Project → Import 该仓库 → Deploy（无需环境变量）。
3. 设密钥：`gh secret set AMAP_KEY` 和 `gh secret set DEEPSEEK_API_KEY`（或 GitHub 仓库 Settings→Secrets）。
4. Actions 页手动 Run 一次 `crawl`，看日志确认抓取/地理编码/研判正常，data 分支生成。
5. 大屏机打开 Vercel 网址，F11 全屏。

## 13. 迁移指南

- **换抓取范围**：改 `zj_live.py` 的 `TARGET`（fid→名称/层级/区）、`MAIN_DISTRICTS`、`ADCODE`，前端 `MAIN_DISTRICTS/ADCODE`，
  补对应 `geojson/<adcode>.json`。改后**重新 seed + 删除并重建 data 分支**（否则旧口径残留）。
- **换平台**：核心是 Python 脚本 + 静态前端，可迁到任意能跑 Python 定时任务的环境（服务器 cron / 云函数）。
  data 分支机制可替换为：① 任意对象存储(OSS/S3) 存 3 个 json，前端改 fetch 地址；② 数据库。`/api/*` 仅是读代理。
- **换地图/底图**：当前用 geojson 多边形(`geojson/` + datav 合并图)。如需真实底图可接 echarts-extension-amap(需高德 JS key+白名单)。
- **境内部署**：若境外抓取被限，把 `zj_live update` 放境内机器定时跑、推 data 分支即可，其余不变。
- **密钥迁移**：高德 REST key、DeepSeek key 重新在新平台配为环境变量/密钥。

## 14. 密钥与安全

- **AMAP_KEY（高德 REST 地理编码）、DEEPSEEK_API_KEY**：存 GitHub Actions 加密密钥，**仅服务端使用，不进代码/前端** ✅。
- **高德 JS key**：曾短暂硬编码做底图，已移除；仍在 git 历史中 → 请到高德控制台**删除/重置**该 JS key。
- 凡 key 在对话或历史中出现过，建议到各控制台**重置**一次更稳妥。
- 若日后用前端高德底图：JS key 无法真正隐藏(必在网页中)，靠**域名白名单**防盗用。

## 15. 故障排查

| 现象 | 排查 |
|---|---|
| 大屏只显示种子数据/不更新 | 看 Actions 是否成功；data 分支 json 的 updated 是否在变；`/api/data` 是否 200 |
| Action 报 403 多 | 境外被限流——调大 `--delay`，或改境内机器跑 |
| 地理编码步骤失败 | 看 AMAP_KEY 是否配；高德额度是否用尽；`geocode.py live` 报错看日志 |
| 风险无研判 | DEEPSEEK_API_KEY 未配 → 只出规则；或额度/网络问题 |
| 线上改了没生效 | Vercel 边缘缓存 + raw CDN 延迟，等 1–3 分钟；前端强刷 Ctrl+Shift+R |
| 地图街道气泡空 | 该区 geo.json 无 streetGeo（地理编码未覆盖）；或 geojson/<adcode>.json 缺失 |

## 16. 调参速查

- 抓取间隔/页数：`zj_live.py update --delay --pages`；定时频率：`crawl.yml` 的 cron。
- 未办结池大小：`zj_live.py` 的 `_trim_low(store["pending"],400)`。
- 风险词库/权重：`risk.py` 的 `RISK_CATS`；DeepSeek 研判数量：`TOP_LLM`；现行阈值：`fresh`=365 天。
- 热词停用词：`zj_live.py` 的 `STOPWORDS`；热词数：`kw` TOP400。
- 轮播节奏：`index.html` 的 `startCar` 的 `5500`(毫秒)。
